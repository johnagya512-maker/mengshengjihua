产品需求文档 (PRD) — 闷声计划
版本：v1.0 | 状态：初稿 | 平台：微信小程序

① 项目概述
项目目的
为自由职业者/独立创业者打造一款极简治愈的 AI 决策执行助手。通过 AI 自动拆解与排期，帮助用户清空大脑焦虑，将复杂项目转化为一个个可执行的具体行动。

解决的核心痛点
痛点	现状	本产品的解法
任务多且乱，不知从哪开始	用户自行在脑中排优先级，高度内耗	AI 自动解析输入，生成带排期的今日任务
传统任务管理工具配置繁琐	用户花大量时间"管理管理工具"	无设置原则，AI 隐形学习用户习惯
长期目标与当日任务脱节	用户失去执行动力，缺乏大局感	进度圆环将每个任务与项目进展可视化关联
突发事件导致计划崩溃	用户放弃当日计划，产生挫败感	每日预留 20% 缓冲 + 弹性重排算法
适用终端
微信小程序（单端），无多端差异。

② 用户角色
使用人群
自由职业者、独立开发者、一人公司创业者。特征：无固定作息、多项目并行、强烈的自我驱动需求。

角色权限
角色	登录方式	可操作功能
普通用户（唯一角色）	微信授权登录（openid 绑定）	全部功能：输入、排期、执行、查看进度、复盘
未登录访客	无	仅可查看引导页，不可使用任何核心功能
无团队/协作/管理员角色，产品定位为"你与 AI 的私人对话"。

  → 弹性重排触发
  → 循环至今日任务清空

④ 功能需求
4.1 微信授权登录
优先级：P0

操作步骤：

用户进入小程序，展示引导页
点击"微信一键登录"按钮
调用 wx.login 获取临时 code
前端将 code 发送至云函数
云函数调用微信接口换取 openid + session_key
以 openid 为主键，查询或创建用户记录
返回自定义登录态 token，前端本地缓存
接口说明：

POST /api/auth/login

请求参数：

参数	类型	必填	说明
code	string	是	wx.login 获取的临时 code
返回字段：

字段	类型	说明
token	string	自定义登录态，有效期 7 天
is_new_user	boolean	是否首次登录，true 则跳转冷启动引导
user_id	string	系统内部用户 ID
状态码：

状态码	含义
200	登录成功
400	code 无效或已过期
500	微信接口调用失败
异常场景：

场景	处理方式
用户拒绝授权	提示"登录后才能使用闷声计划"，保留重新登录入口
网络异常	提示"网络开小差了，请重试"，按钮恢复可点击
微信接口超时	同上，前端超时阈值 5s
token 过期	静默刷新，用户无感知；刷新失败则跳转登录页
4.2 冷启动引导（Onboarding）
优先级：P0

触发条件： is_new_user = true 时自动触发，仅首次执行一次。

操作步骤：

进入引导界面，AI 以对话气泡形式依次提问
问题 1：专注高峰时段（时间段选择器，可多选）
问题 2：最头疼的任务类型（标签多选，选项由 AI 预设）
问题 3：理想每日工作时长（滑动条，1h ~ 12h，默认 6h）
用户完成全部选择后，点击"开始闷声"进入主界面
将 3 项数据写入用户 Profile，作为排期算法初始种子
接口说明：

POST /api/user/profile/init

请求参数：

参数	类型	必填	说明
peak_hours	array<string>	是	精力高峰时段，如 ["09:00-11:00", "15:00-17:00"]
pain_task_types	array<string>	是	头疼任务类型标签，至少选 1 个
ideal_work_hours	number	是	理想工作时长，单位：小时，范围 1~12
返回字段：

字段	类型	说明
success	boolean	写入成功
异常场景：

场景	处理方式
用户未完成全部问题直接关闭小程序	下次进入重新从引导开始，Profile 未初始化则不进入主界面
提交失败（网络异常）	本地暂存，网络恢复后自动重试，不阻塞用户进入主界面
4.3 任务输入（Capture）
优先级：P0

操作步骤：

主界面底部常驻"大按钮"（全局唯一入口）
点击后展开输入面板，支持两种输入方式：
文字输入：自由文本框，支持长文本
语音输入：长按麦克风按钮，调用微信原生 wx.getRecorderManager，松开后自动转文字
用户提交后，前端调用 AI 解析接口
AI 返回解析结果，以"确认卡片"形式展示给用户
用户可修改 AI 的解析结果，确认后写入任务队列
AI 解析输出的四项结果：

字段	说明	示例
action	任务目标，动词开头，一句话	"完成落地页设计初稿"
duration	预估耗时	15 / 30 / 45 / 60（分钟）
project_tag	关联项目名称，AI 自动匹配或新建	"品牌升级"
vision_statement	愿景引导文案，激励性短句	"迈出这一步，项目就活了"
接口说明：

POST /api/task/parse

请求参数：

参数	类型	必填	说明
input_text	string	是	用户原始输入文字
user_id	string	是	用户 ID
返回字段：

字段	类型	说明
action	string	解析后的任务目标
duration	number	预估耗时（分钟）
project_tag	string	关联项目名
vision_statement	string	愿景文案
is_new_project	boolean	project_tag 是否为新建项目
状态码：

状态码	含义
200	解析成功
422	输入内容无法识别为任务（如纯数字、空内容）
500	AI 接口调用失败
504	AI 接口超时（阈值 8s）
异常场景：

场景	处理方式
AI 解析超时（>8s）	提示"AI 在想，稍等一下"，超过 8s 提示"换个方式试试"，给出重试按钮
AI 无法识别输入内容	返回提示"没太理解，能再说清楚一点吗？"，不生成任务卡片
语音转文字失败	提示"没录到声音"，保留文字输入兜底
用户关闭确认卡片	任务不写入队列，输入内容不保留
网络中断时提交	本地暂存输入内容，网络恢复后提示用户"有一条任务还没处理"
4.4 动态决策引擎（Decision Engine）
优先级：P0（基础排期）/ P1（弹性重排）

排期规则（P0）：

以用户 Profile 中的 peak_hours 作为高优任务时间窗口
P0 级任务（用户标记或 AI 判断为紧急）优先填入高峰时段
每日总可用时长 = ideal_work_hours × 60 × 80%（预留 20% 缓冲）
超过当日容量的任务自动移至次日
疲劳补偿规则：

单次连续任务耗时 ≥ 90 分钟，自动在其后插入 10 分钟"治愈间隙"
治愈间隙在任务列表中以特殊样式展示，不可删除，可跳过
弹性重排规则（P1）：

任务完成/跳过后，后台在 2s 内完成剩余任务重排
重排考虑因素：当前时间、剩余时长、任务紧急度、用户历史跳过记录
重排结果静默更新，无需用户感知
接口说明：

POST /api/schedule/compute

请求参数：

参数	类型	必填	说明
user_id	string	是	用户 ID
trigger	string	是	触发原因：add_task / complete / skip / daily_init
task_id	string	否	触发任务 ID（complete/skip 时必填）
返回字段：

字段	类型	说明
ordered_tasks	array	重排后的任务列表，含 task_id 和 scheduled_time
overflow_tasks	array	超出当日容量、移至次日的任务列表
daily_capacity_used	number	当日已用时长（分钟）
daily_capacity_total	number	当日总可用时长（分钟）
异常场景：

场景	处理方式
重排计算超时（>2s）	保持当前顺序不变，后台继续计算，完成后静默更新
当日无任务可排	主界面显示"今天清空了，牛。"
4.5 任务执行（Execution）
优先级：P0（确认卡片）/ P1（耗时滑动微调）

操作步骤：

主界面展示当前最高优任务卡片
用户点击"开始"，弹出确认卡片，包含：
愿景引导文案（vision_statement）
预估耗时展示
耗时微调滑动条（P1）：左滑减少 15min，右滑增加 15min，范围 15~120min
用户确认后进入专注计时界面
计时结束后弹出完成确认，用户标记"完成"或"跳过"
触发弹性重排（见 4.4）
状态反馈 — 进度胶囊：

常驻于主界面顶部
显示内容："今日剩余可支配创造力：X 小时 X 分钟"
数值 = 当日剩余可用时长（含缓冲扣除）
接口说明：

POST /api/task/complete

请求参数：

参数	类型	必填	说明
task_id	string	是	任务 ID
actual_duration	number	是	实际耗时（分钟）
result	string	是	complete / skip
skip_reason	string	否	跳过原因，result=skip 时必填
返回字段：

字段	类型	说明
success	boolean	写入成功
next_task_id	string	下一任务 ID
异常场景：

场景	处理方式
计时中途退出小程序	后台记录中断时间，重新进入时询问"任务做完了吗？"
提交完成状态失败	本地缓存结果，网络恢复后自动同步，不影响用户继续下一任务
4.6 进度圆环可视化（Project Rings）
优先级：P2（MVP 内）

展示规则：

每个项目对应一个圆环
圆环进度 = 已完成任务数 / 项目总任务数
圆环颜色与项目标签绑定，低饱和度治愈色系，系统自动分配，不可手动修改
项目圆环入口：主界面底部"项目"Tab
任务点流（Task Dots）：

点击圆环，展开项目内部任务列表
每个任务以圆点形式展示（完成=实心，未完成=空心，跳过=半透明）
不展示复杂逻辑，只展示任务名 + 状态
接口说明：

GET /api/project/list

返回字段：

字段	类型	说明
projects	array	项目列表
projects[].project_id	string	项目 ID
projects[].name	string	项目名
projects[].color	string	圆环颜色 HEX 值
projects[].total_tasks	number	总任务数
projects[].completed_tasks	number	已完成任务数
projects[].tasks	array	任务点列表
⑤ 字段规则
字段名	类型	是否必填	长度限制	格式校验	错误提示文案
input_text（任务输入）	string	是	1~500 字符	非纯空白字符	"说点什么吧"
action（任务目标）	string	是	1~100 字符	非空	—
duration（预估耗时）	number	是	—	枚举值：15/30/45/60/75/90/105/120	"请选择耗时"
project_tag（项目名）	string	是	1~30 字符	非空，不含特殊符号	"项目名不能为空"
vision_statement（愿景文案）	string	是	1~80 字符	非空	—
peak_hours（高峰时段）	array	是	至少 1 项	格式 HH:mm-HH:mm	"请选择你的专注高峰"
ideal_work_hours（理想工作时长）	number	是	—	范围 1~12，步进 0.5	"请设置每日工作时长"
skip_reason（跳过原因）	string	条件必填	—	枚举：没状态/等待外部/临时取消	"请选择跳过原因"

性能要求
指标	要求
首页加载时间	≤ 1.5s（本地缓存命中），≤ 3s（冷启动）
AI 任务解析响应	≤ 5s（正常），≤ 8s（超时阈值，触发提示）
弹性重排计算	≤ 2s
接口通用响应	≤ 1s（非 AI 接口）
并发量	MVP 阶段支持 500 DAU，峰值 100 并发
兼容性要求
维度	要求
微信版本	支持微信 8.0.0 及以上
操作系统	iOS 14+ / Android 8+
屏幕适配	兼容 375px ~ 428px 宽度（iPhone SE ~ Pro Max）
深色模式	跟随系统，治愈色系需在深色模式下验证不失真
安全约束
项目	要求
鉴权方式	微信 openid + 自定义 token，token 有效期 7 天，每次请求 Header 携带
敏感数据	用户任务内容属于隐私数据，云数据库按 openid 隔离，不同用户间数据绝对隔离
AI 接口	DeepSeek/GPT API Key 存储于云函数环境变量，不得暴露在前端代码中
语音数据	语音文件仅用于本次 ASR 转换，转换完成后立即丢弃，不做持久化存储
输入过滤	所有用户输入在传入 AI 前做基础过滤（SQL注入、XSS），防止 Prompt Injection

⑧ 本次不做的功能
以下功能明确排除在 MVP 范围之外，防止后期加需求：

团队协作 / 任务共享：产品定位为单用户私人助手，不做任何协作功能
星系复盘视图（Galaxy View）：长期目标层级关系的可视化，列为 P3
历史精力数据分析：周/月维度的个人效率报告，列为 P3
自定义项目颜色/图标：圆环颜色由系统自动分配，用户不可手动修改
任务提醒推送（微信订阅消息）：MVP 不做主动推送，用户自行进入小程序查看
任务评论/备注：每条任务只有 action + vision，不支持附加笔记
多设备实时同步冲突处理：云端以最后写入为准，不做复杂冲突解决
离线模式完整功能：网络断开时只读本地缓存，不支持离线新增任务
