# 隐形学习 + AI 主动能力 设计文档 — 闷声计划

> 配套 PRD v1.0。本文档定义「AI 隐形学习用户习惯」的具体规则、触发时机、计算方式、护栏，
> 以及由它驱动的三个 AI 主动能力：任务拆分建议、每日完成跟踪、教练式问责反馈。

## 0. 总原则（所有规则的灵魂）

**校正，不评判。问责是好的，但必须自主、支持，不强加、不惩罚。**

核心用户是「缺乏外部结构、需要自我驱动」的人——自由职业者是典型，但也含学生、
创作者、副业者、公司里想自己管自己的人。正因为没有老板提供节奏兜底，这类人
**比上班族更需要**时间安排的结构与问责，而不是更少。

所以本产品不回避「盯」，而是讲究这份盯的**来源与口吻**：

| 反对（老板式监督） | 推崇（教练式问责） |
|------------------|------------------|
| 被强加、怕被罚、为别人达标 | 自己默许、为自己负责、有人陪你扛 |

差别不在「有没有人看着」，而在**这份问责是用户主动请来的、口吻是支持而非惩罚的**。
健身教练也盯你，但你花钱请他盯——同样是盯，你感激他。承诺装置（commitment device）
在行为经济学里被反复验证：人主动给自己设约束，反而更自由、更能成事。

隐形学习只做一件事：观察用户真实的样子，然后服务他——既不放任，也不充当
"你今天少做了"的记分员。下文每条规则、每句文案，都受此约束。

---

## 1. 为什么要隐形学习（定位）

学习不是为了实时调今天的排期，而是**攒一份"对你的理解"**。
等用户定项目目标时，系统才能给出符合个人情况的建议
（"按你最近的节奏，这个项目每天投 X 小时、排 Y 时段最推得动、预计 Z 天完成"）。

闭环：
```
定项目目标 →（用学到的参数）给建议 → 用户执行
   → 行为数据回流 → 学习校正参数 → 下个项目建议更准
```
越用越懂你。这是冷启动删掉引导问卷后，个性化的唯一来源。

## 3. 五条学习规则（每条都绑一个原理）

### 规则 1　精力高峰：用「完成行为」反推，不信自报
- **信号**：按时段统计——完成率高 + 跳过率低 + 实际耗时≈预估的时间窗，即真高峰
- **原理**：揭示性偏好 > 陈述性偏好。人嘴上说不准几点专注，行为不会骗人。
  （这是当初敢删掉「问你几点专注」的底气——该让数据回答）
- **时间管理**：把高价值任务排进真高峰 = 在对的时间吃掉那只青蛙

### 规则 2　每日容量：治「规划谬误」
- **信号**：过去 2~3 周实际完成总时长的**滚动中位数**（非平均，抗异常值）
- **原理**：Kahneman 规划谬误——人系统性高估自己。自报 6h，实际可能 4h
- **人性（关键）**：与其每天「没做完」而内疚，不如把容量调到真实能做到的量，
  让用户**每天都能清空**。成就感而非挫败感——治愈系与普通待办的分水岭

### 规则 3　耗时预估：学一个个人校正系数
- **信号**：实际/预估 比值，按任务类型分桶（写代码 vs 写文案，偏差不同）
- **原理**：参照类预测（Flyvbjerg）——用自己的历史当参照系，比拍脑袋准
- **时间管理**：预估越准，排期越不崩，弹性重排越少触发

### 规则 4　治愈间隙节奏：超日节律个人化
- **信号**：连续工作多久后，跳过率 / 「没状态」开始抬头
- **原理**：超日节律 ~90min 是平均值，个体差异大。focus_tolerance 参数化即为此
- **人性**：在真正累之前插间隙，**预防**疲劳，而非垮了再补救

### 规则 5　跳过归因：区分「你的问题」和「环境的问题」（最有同理心）
| 跳过原因 | 系统理解 | 处理 |
|---------|---------|------|
| 没状态 | 能量错配 | 这类任务往真高峰挪 |
| 等待外部 | 被依赖阻塞，**不是你的错** | 不降权、不算失败，标记「等待中」 |
| 临时取消 | 优先级变了，正常 | 不惩罚 |
- **原理**：归因理论。若系统把「等待外部」当成执行力差，用户会觉得被冤枉。
  分不清「你失败了」和「情况变了」的工具，用久了让人厌恶

---

## 4. 护栏（比规则更重要——学坏比不学还糟）

1. **冷启动门槛**：完成记录 < 20 条不学，先用默认。避免头一两天噪声「定终身」
2. **滚动窗口 + 中位数**：只看最近 2~3 周。生病那一周不该重新定义你是谁
3. **探索 vs 利用**（最易被忽略）：不能因为「学到你早上常跳过」就再也不排早上——
   那是自我实现的死循环，你永远没机会证明自己变了。保留约 15% 探索性排期
4. **永远可被覆盖**：学习只调种子参数，用户当下手动调整优先级最高
5. **校正而非加压**：学到你能做更多 ≠ 就给你排更多。容量上限为**可持续**服务，
   不为榨取产出。这条踩了，整个产品就背叛了「闷声 / 治愈」

## 5. 由学习驱动的三个 AI 主动能力

### 5.1 任务拆分建议（task_parse 升级）
- **触发**：解析时识别「大事」——预估 >90min，或措辞含「项目 / 方案 / 搞定XX / 上线」
- **行为**：拆成 2~5 个可执行小步，每步是动词开头的一句话
- **学习反哺**：拆分粒度参考用户的 `focus_tolerance`——专注力强的拆得粗，反之拆细
- **护栏**：拆分是**建议**，确认卡片里用户可改可删可合并；不强制

### 5.2 每日完成跟踪（数据源 + 问责 + 回响）
- **本质**：既是隐形学习的数据采集，也是用户要的「结构感 / 被盯着」
- **呈现**：进度胶囊「今日剩余创造力」+ 当日进度（已清掉 X / 共 Y）
- **主动提醒**：可以有——这正是自我驱动人群需要的外部结构。区别只在口吻（见 5.3）
- **晚间收束**：当日首次空队列时收尾——「今天清空了，牛。」（PRD 已有）

### 5.3 教练式问责，不是老板式监督（这条定生死）

核心用户缺乏外部结构、要自我驱动，**比上班族更需要问责**——没有老板兜底，松掉就散了。
所以本产品不回避「盯」，承诺装置（commitment device）是行为经济学验证有效的手段。
关键不在「要不要盯」，而在这份盯的**来源与口吻**：

| 老板式监督（错） | 教练式问责（对） |
|----------------|----------------|
| 被强加、为别人达标 | 用户自己默许、为自己负责 |
| 「你又没做到」——评判 | 「这件卡住了？要不要挪到状态好的时候」——帮你解决 |
| 罚你 | 陪你扛 |

设计准则：
- **主动提醒可以有**（用户要的结构感），但口吻是教练/队友，不是上司
- **负向计数（「还差 X 件」）可以出现**，但永远配「帮你解决」的姿态——
  紧跟一个动作建议（挪时段 / 拆小 / 降优先级），而不是单纯施压
- **跳过不评判**：按归因（规则5）回应——「等待外部」标记等待中、绝不算失败；
  「没状态」提议挪到高峰；从不追问「为什么没做」
- **提醒频率自适应**：可被关闭，且学习用户对提醒的反应（总是忽略 → 自动减少）
- 口吻：教练、队友——既盯着你，也站在你这边

## 6. 实现：profile_learn 云函数

- **触发**：每日定时（或用户当天首次登录时）跑一次，静默执行
- **输入**：tasks / skip_logs 近 3 周数据
- **输出**：更新 profiles 的 peak_hours / ideal_work_hours / focus_tolerance，
  并写一份 learning_meta（学习依据、样本量、置信度），便于调试与解释
- **完全无感知**——这才叫「隐形」

### 字段新增
| 集合 | 字段 | 说明 |
|------|------|------|
| profiles | duration_bias | 各任务类型的实际/预估系数（规则3） |
| profiles | learning_meta | 最近学习时间、样本量、是否仍用默认 |
| tasks | est_vs_actual | 完成时落盘，供规则3统计 |

## 7. 双向闭环（回应「定项目时给科学建议」）

```
定项目目标 →（用学到的参数）给「每天X小时 / 排Y时段 / 预计Z天」建议
   → 用户执行 → 行为回流 → profile_learn 校正 → 下个项目建议更准
```

「定项目目标」是 PRD 之外的新功能，与本学习系统构成闭环，建议两者一起规划
（当前 PRD 的项目是 task_parse 解析时自动建，无显式定目标步骤）。

## 9. 督学模式（会员功能 · P2 · 仅设计，暂不开发）

### 9.1 为什么做
- **已验证的付费需求**：小红书等平台「真人监督学习」已售出数万份，付费意愿被市场证明，
  非臆测需求 → 不算过度开发
- **天然契合「自主问责」框架**：用户写任务时**自己勾选**「需要监督」，正是教科书级的
  承诺装置（commitment device）——用户主动请来的约束
- **商业模型合适**：AI 驱动的监督边际成本低、可规模化、24h 在线，相对真人督学是降维

### 9.2 形态
- **入口**：写任务时可选「开启监督」；或对某个重要任务单独开启
- **默认关闭**：保住「治愈 / 闷声」基调，监督是按需召唤，不是常驻
- **随时可关**：控制权始终在用户手里 → 是「按需的教练」，不是「甩不掉的老板」
- **能力**：定时检查进度、按用户习惯推送督促、完成确认、阶段性复盘鼓励

### 9.3 免费 / 会员边界
| | 免费版 | 会员版 |
|--|--------|--------|
| 定位 | 工具：帮你理任务、排日程 | 教练：盯着你、督促你完成 |
| 含 | 任务输入 / AI解析 / 排期 / 执行 / 项目圆环 | + 督学模式 + 更主动的问责 |

### 9.4 三个坑（开发前必须解决）
1. **顺序**：必须在「核心跑通 + 隐形学习成熟」之后做。顺序错=过度开发，顺序对=商业化
2. **必须有「人感」**：没有隐形学习支撑的督学 = 贵闹钟。督促时机/口吻/接话都要用学习
   到的用户习惯。**督学是隐形学习长出来的果实，不是独立功能**
3. **不与治愈定位打架**：督学本质是加压，靠「用户自己请来的 + 随时可关 + 教练口吻（规则5归因）」
   化解。压力必须是自主的、可控的

### 9.5 依赖
督学模式依赖：规则1（高峰，知道何时督促有效）、规则5（跳过归因，督促口吻）、
5.3（教练式问责文案）。这些 P1/P2 完成后，督学才具备「人感」，故置于 P2 末、暂不开发。
