# 真机测试清单

> 我(AI)已帮你检查的项,标 ✅;只有你能在开发者工具/手机/云控制台做的,标 ⬜ 待你操作。

## 一、上手机前的必做项(不做会直接报错)

### ⬜ 1. 配置 DeepSeek API Key(最容易忘,忘了拆解必失败)
AI 拆解靠环境变量 `DEEPSEEK_API_KEY`,代码里没硬编码(安全做法对)。
操作:微信开发者工具 → 云开发控制台 → 云函数 → `task_parse` → 配置 → 环境变量,
新增 `DEEPSEEK_API_KEY = 你的 key`。
- 注意:`profile_learn`/`task_save` 等不调 AI 的云函数不需要这个 key。
- 验证:配完后在工具里调一次"交给 AI 拆解",看是否返回步骤。

### ⬜ 2. 部署所有改过的云函数(真机连云端,不是本地代码)
本轮改过、必须重新「上传并部署」的:
- `profile_patch`(加了手动容量标记)
- `profile_learn`(加了不覆盖手设容量的护栏)
操作:开发者工具 → 右键云函数文件夹 → 上传并部署(云端安装依赖)。
- 提示:`task_parse` 等云函数本地 node_modules 未装,但「上传并部署(云端安装依赖)」会在云端装,无需本地 npm install。

### ✅ 3. mock 已关闭
`request.ts` 里 `USE_MOCK = false`,真机会走真实云函数。无需操作。

### ⬜ 4. 确认加载的是修复分支
本轮修复在 `fix/learning-trigger-and-id` 分支,尚未合并到 main。
真机测试前确认开发者工具打开的是这个分支的代码(或先合并)。

## 二、真机测试重点用例(带着问题测,别瞎点)

### A. 核心循环(产品成不成立的关键)
- [ ] 打开"大事帮我拆成几步"开关 → 输入"写完季度报告" → AI 是否拆解?步骤合理吗?
- [ ] 第一步是否"小到不可能拒绝"(2~5分钟能做完)?
- [ ] 不开开关 → 输入一句话 → 是否秒记一条(快速添加)?

### B. 隐形学习(本轮修复重点,务必验)
- [ ] 完成 1 个任务后,去云开发控制台看 `profiles` 表:
      `learning_meta` 和 `learned_at` 有没有更新?(验证触发活了)
- [ ] 同一天再完成任务,`learned_at` 应不变(验证当天只学一次的节流)
- [ ] 点进度胶囊手动改"每日可投入时长" → 看 `profiles` 表
      `ideal_work_hours_manual` 是否为 true(验证手设标记)

### C. quickAdd id 修复
- [ ] 快速添加一条 → 立刻(趁排期还没回来)对它点完成/删除 → 操作是否生效?
- [ ] 飞行模式下快速添加 → 恢复网络 → 数据是否正确同步、不重复?

### D. 真机特有的坑(模拟器测不出)
- [ ] 专注计时中锁屏/切后台 1 分钟再回来,计时准不准?
- [ ] 弱网(手机信号差或限速)下,输入/完成的体验
- [ ] 不同机型/屏幕尺寸下样式是否错位
- [ ] 完成任务后从专注页返回首页,任务状态是否正确(不复活已完成)

## 三、已知未修问题(测试时会撞到,心里有数)
- review 复盘页"今日/本周"视图的生涯卡数字偏小/为 0(14天取数窗口截断)

- AI 未设 max_tokens,异常超长输入成本不可控
