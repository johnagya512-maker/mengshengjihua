# MVP 设计与范围 — 闷声计划

> 配套 PRD v1.0 与《隐形学习设计》。本文档钉死「第一个可上线版本」实际要做什么，
> 记录相对原始 PRD 的**决策变更**，作为开发的唯一范围依据。范围之外的一律不做。

## 0. 一句话定义

> 说一句话 → AI 拆成可执行任务 → 自动排进今天 → 专注完成 → 项目自然推进。
> 无设置、无问卷、秒进主界面。

核心用户：缺乏外部结构、需要自我驱动的人（自由职业者为典型，也含学生 / 创作者 / 副业者）。

---

## 1. 相对原始 PRD 的决策变更（重要）

| # | 原始 PRD | MVP 实际 | 原因 |
|---|---------|---------|------|
| 1 | 冷启动 3 问引导 | **删除引导**，首登静默写默认 Profile | 兑现「无设置原则」，降流失 |
| 2 | 引导问「专注高峰」 | 改由隐形学习从行为反推（P2） | 用户答不准，行为不骗人 |
| 3 | 引导问「头疼任务类型」 | **删除**（算法未消费，假需求） | 收集了不用 = 浪费 |
| 4 | 引导问「每日工时」 | 默认 6h 种子，留待「定项目目标」匹配 | 工时属项目级参数，问太早 |
| 5 | 6 个云函数 | 11 个（加 task_save 写库、profile_learn 隐形学习、task_delete 删留痕等） | 随功能演进新增 |
| 6 | — | 默认 Profile：双峰高峰 / 专注45 / 工时6 | 删引导后的安全起点 |

## 2. MVP 范围内（要做）

### 2.1 已完成（代码 + 验证）
- 微信 openid 登录，首登静默建用户 + 默认 Profile
- 任务输入 → AI 解析（task_parse / DeepSeek）→ 确认卡片 → 写队列
- 排期引擎：80% 缓冲、高峰优先、容量饱和移次日（算法测试覆盖）
- 专注计时：耗时微调、完成 / 跳过 + 归因、中断恢复
- 项目圆环 + 任务点流
- 离线暂存（输入 / 完成）自动回放
- 语音输入（插件未授权时优雅降级）

### 2.2 MVP 仍需补（开发中）
- 隐形学习 P1：规则2容量 / 规则3耗时 / 规则5跳过归因
- 任务拆分建议（task_parse 识别「大事」拆小步）
- 教练式问责文案（完成回响 + 支持性提醒）
- 隐私政策入口（登录页小字 + 链接，上线合规要求）

---

## 3. MVP 范围外（明确不做，防跑偏）

| 功能 | 放到 | 为什么不进 MVP |
|------|------|--------------|
| 治愈间隙 / 疲劳补偿（≥90min 自动插 10min gap） | P2 | 排期引擎暂未实现；常量 FATIGUE_THRESHOLD/HEAL_GAP_DURATION 已预留但未启用 |
| 隐形学习 P2：高峰反推、间隙节奏个人化 | P2 | 需 2~3 周行为数据才有意义 |
| profile_learn 定时学习 + 探索/利用护栏 | P2 | 同上 |
| 督学模式（会员功能） | P2 | 依赖学习成熟，且需先验证核心 |
| 定项目目标 + 项目级建议 | P3 | 闭环功能，PRD 外新增 |
| 历史精力分析 / 数据报表 | P3 | PRD 已排除出 MVP |
| unionid 跨小程序统一身份 | 暂不 | 单小程序内不需要 |

原则：**MVP 只验证「核心循环成不成立」**——用户愿不愿意用"说话→拆解→排期→完成"。
学习与商业化都建立在核心被验证之后，提前做就是过度开发。

## 4. 页面与流程（MVP 实际形态）

```
guide（登录）──登录即进──> home（今日）
                              │  ├─ 大按钮 → 输入面板 → AI解析 → 确认卡片 → 入队
                              │  └─ 任务卡「开始」→ focus（专注计时）→ 完成/跳过 → 回 home
                              └─ tabBar 切 → projects（项目圆环）
```
- 4 个页面：guide / home / focus / projects（onboarding 已摘除路由，文件保留）
- 无引导页、无设置页、无个人中心（MVP 不需要）

## 6. 验收（MVP 通过标准）

对应 PRD 的 TC，MVP 必须跑通：
1. 登录直进主界面（无引导）
2. 说一句话 → 出确认卡片 → 入队 → 主界面出现任务卡 + 容量变化
3. 开始专注 → 计时 → 完成 → 下一条
4. 跳过 + 选原因 → 写 skip_logs
5. 项目 Tab 圆环正确反映完成度
6. 杀进程重进 → 数据完整（云端按 openid 恢复）
7. 断网输入/完成 → 联网后自动回放
