# 项目三态设计（有限 / 结果 / 持续）

## 背景与问题

现状：项目进度只有一个公式 `percent = completed_tasks / total_tasks`（见 `pages/projects/projects.ts`）。

这个公式只对**有终点的项目**成立（"读100本书"）。对**持续运营型项目**（"小红书"）会算出假指标：
- 进度随新任务加入而"倒退"，越投入圆环越不满，**惩罚了最活跃的项目**。
- 圆环永远走不满 = 持续的焦虑源，违背"闷声计划"低负担的内核。

根因不在"归类"，在**量化方式单一**。不同项目要用不同的"完成感"来度量。

## 三种项目形态

| 模式 | mode 值 | 例子 | 量化锚 | 进度来源 | 会倒退？ |
|------|---------|------|--------|----------|----------|
| 有限计数 | `count` | 读100本书、发100条笔记 | 完成数 / 目标数 | 任务自动累加 | 否，只涨 |
| 结果目标 | `result` | 月入一万、涨粉到1万 | 当前值 / 目标值 | **用户手动记** | 可能（周期重置） |
| 持续投入 | `streak` | 长期运营小红书 | 连续天数 / 累计件数 / 本周件数 | 任务自动算 | 不设总进度 |

核心区别：
- **有限**问"还差多少" → 进度条走向 100%，可结业。
- **结果**问"离目标还差多少" → 数值是结果而非动作，系统推不出，必须手动记；按周期结算重置。
- **持续**问"最近有没有在动" → 要的是"心跳/势头"，不是进度条。

## 数据结构变更

`projects` 集合现有字段：`_openid / project_id / name / color / created_at`。新增：

| 字段 | 类型 | 说明 |
|------|------|------|
| mode | string | `count` / `result` / `streak`，默认 `count` |
| goal_target | number? | 目标值。count=目标件数（如100），result=目标数值（如10000）。streak 不用 |
| goal_unit | string? | 结果型单��，如「元」「粉丝」。仅 result |
| cycle | string? | 结果型结算周期：`month` / `week` / `none`。仅 result |
| current_value | number? | 结果型当前进度，用户手动记。仅 result |
| current_value_at | number? | current_value 最近更新时间，用于周期重置判断 |

`随手记` 兜底项目固定 `mode: 'count'` 且不设 `goal_target`（无目标，只统计件数），是收件箱，不参与"结业"。

## 三种项目卡片

count（有限计数）—— 沿用现有圆环：
- 圆环 = `completed_tasks / goal_target`（注意分母从 `total_tasks` 改为 `goal_target`）
- 中心显示百分比，下方 `已完成 N / 目标 M`
- 走满 100% = 结业态，圆环变金色 + 「已达成」标记

result（结果目标）—— 不用圆环，用数值进度条：
- 进度条 = `current_value / goal_target`，显示 `¥3200 / ¥10000`
- 卡片带「记一笔」按钮 → 弹输入框更新 `current_value`
- 显示周期：「本月目标」，月底结算后 `current_value` 归零、`current_value_at` 重置
- 配合显示坚持指标（见下），避免月初进度为 0 时卡片空荡

streak（持续投入）—— 无总进度，显示势头：
- 主指标「已坚持 N 天」（连续有完成任务的天数）
- 副指标「本周 N 件 / 累计 M 件」
- 断更不报错，只是天数归零，不制造焦虑文案

## 归类与 AI 边界

- 模式由**用户建项目时手动选**，不调 AI。理由：低频、强主观、AI 推不出 goal_target，选完还得用户确认，不划算。
- AI 只用在**首页加任务时归类到已有项目**（高频、客观、省心），不用在建项目环节。
- 建项目入口：项目页加「＋ 新建项目」→ 选模式 →（count/result 填目标）→ 调 `api.createProject`。
  需扩展 `project_create` 云函数与 `createProject` 接口，接收 mode / goal 相关字段。

## 落地顺序（建议）

1. 后端：`project_create` 接收并存 mode/goal 字段；`project_list` 返回这些字段。
2. 前端：项目页「＋ 新建项目」入口 + 模式选择面板。
3. 前端：`projects.ts` 按 mode 分流渲染三种卡片；count 圆环分母改 `goal_target`。
4. result 模式「记一笔」+ 周期重置逻辑（可后置）。
5. streak 坚持天数 / 累计件数统计（依赖任务完成时间，可后置）。
